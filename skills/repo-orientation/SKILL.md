# リポジトリ初期把握

## 目的

初見のリポジトリをすばやく把握し、安全に変更を進めるための文脈を整える。

## このスキルを使う場面

- まだ調査していないリポジトリで支援を求められたとき
- 編集前に構成、ツール、ローカル規約を理解する必要があるとき
- 適切な入口ファイルや主要コマンドを早く見つける必要があるとき

## このスキルを使わない場面

- 依頼が概念的な相談だけで、リポジトリの状態に依存しないとき
- 現在のセッションですでに十分なリポジトリ文脈が確立しているとき

## 入力

- 現在の作業ディレクトリ
- ユーザーからのタスク説明
- `AGENTS.md`、`README.md`、ツール設定などのリポジトリ指示

## 手順

1. リポジトリのトップレベル構成を確認する。
2. 重要な指示ファイルと README を読む。
3. 主なソースディレクトリ、スクリプト、テスト入口を特定する。
4. 編集、検証、リリース作業に影響する制約を整理する。
5. 深い作業に入る前に、必要最小限の把握結果を共有する。

## 出力

- 短いリポジトリ構成の要約
- タスクに重要なコマンドやファイル
- 直近のリスク、未セットアップ事項、未確定の前提

## ガードレール

- 要約で足りる場面では大量の生出力をそのまま並べない
- 確認せずにフレームワークやパッケージマネージャーを決め打ちしない
- リポジトリが疎だったり未完成だったりする場合は不確実さを明示する

## 依頼例

「このリポジトリを見て、共有スキルをどこに追加すべきか整理して」

## 結果例

`skills/` が正式な置き場だと示し、テンプレートの使い方と公開可能なスキルとして追加する流れを説明する。
