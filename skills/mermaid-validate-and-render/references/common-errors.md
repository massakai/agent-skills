# Common Errors

## parse エラー

### ノードやエッジの構文崩れ

- 症状: 矢印や括弧の近くで parser が失敗する
- 対処: まず `A --> B` のような最小の有効エッジに戻し、その後でラベルを足し直す

### 危険または不整合な ID

- 症状: parse が失敗する、または後続参照が一致しない
- 対処: `service_api` のような単純な ASCII ID に正規化し、表示名はラベル側へ寄せる

### ラベルの引用ミス

- 症状: 記号、括弧、コロン付近で parser が失敗する
- 対処: 記号を明示的なラベルに移し、ノード ID は単純に保つ

### `subgraph` ブロックの不整合

- 症状: `subgraph` や `end` の近くで parse が失敗する
- 対処: すべての `subgraph` に対応する `end` があることを確認し、ラベルを単純化してから再検証する

## render エラー

### parse は通るのに renderer が失敗する

- 症状: `validate_mermaid.mjs` は成功するが `mmdc` が失敗する
- 対処: 直前に追加した高度な構文を外し、`parse` から再開する

### 未対応または壊れやすいスタイル指定

- 症状: 複雑なスタイルや class 追加後に render が失敗する
- 対処: `classDef`、インライン style、theme 設定を一度外し、まず素の図を描画させる

### レイアウト破綻

- 症状: render は通るが図が崩れる、極端に読みにくい
- 対処: ラベルを短くする、交差を減らす、複数図へ分割する

## 復旧パターン

1. 失敗する最小例まで図を縮める
2. `parse` 成功を回復する
3. `render` を再実行する
4. 詳細は一段ずつ戻す
